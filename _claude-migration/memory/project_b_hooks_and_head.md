### `useContext` — DONE 2026-06-27 (⚠ UNCOMMITTED) — HOOK FAMILY NOW COMPLETE
- **Client (Tier-C, full + correct):** `createContext(default)` returns `{__albedoContext, _id, _defaultValue, Provider}`; `Provider.__albedoContextId = id`. Provider is a **Fragment-style sentinel** special-cased in all three walkers (`instantiate`/`hydrateInstance`/`reconcile`) — it does NOT run a component body; it renders its single child (so `singleFragmentChild` applies → multi-child Provider is the same deferred gap as Fragment) under a pushed `currentContextMap` (contextId→provider *instance*). Each component instance **snapshots** `contextMap` at mount; `useContext` reads `currentFiber.contextMap[ctx._id]` and returns the provider instance's **live** `vnode.props.value` (no positional hook slot consumed). Consequences proven: a deep consumer re-rendering on its OWN state still resolves context (snapshot, not a transient stack); a Provider value change held in ancestor state propagates on the ancestor's reconcile (provider vnode refreshed first, child reconciled under updated map). `unmount` walks `providerChild`. Exported on `__albedoClient` + globals.
- **SSR (Tier-B, non-crashing):** `globalThis.createContext`/`useContext` added to the QuickJS hook prelude (`src/runtime/quickjs_engine.rs`, in the `if (typeof globalThis.useState !== 'function')` block). `useContext` returns `host.context[id]` seed if present else `ctx._defaultValue`; Provider renders `props.children` (or `''`). Wired through framework-import rewrite: named `import { useContext, createContext } from "react"` resolve to globals automatically; namespace/default via additions to `FRAMEWORK_NAMESPACE_OBJECT`.
- **Documented limit (honest):** SSR `h` invokes components **eagerly** (line ~638) — children are already-rendered HTML strings before a Provider runs — so a nested Provider can't thread its value down in the single server pass. Server-side a consumer resolves the **createContext default** (or host seed); the client applies the real value on hydration → possible first-paint mismatch when provider value ≠ default. True SSR threading needs lazy element trees (defer). Tiering already classifies `useContext(` (`src/runtime/static_slice.rs` `STATIC_SLICE_HOOK_MARKERS`).
- **Tests:** `tests/client_hydration.rs::client_runtime_resolves_usecontext_through_provider` (3 invariants, zero network) + `quickjs_engine::tests::use_context_renders_default_without_crashing`. All green: 406 lib + 8 reactive_bindings + 6 client_hydration. Files touched: `assets/albedo-client.js`, `src/runtime/quickjs_engine.rs`, `TODO.md`.

Fixture (kept, portfolio is no-git): `albedo-portfolio/src/components/ThemeToggle.tsx` (self-contained island: `createContext`, a Provider holding `useState("dark")`, a pure `ThemeLabel` consumer, a `ThemeReadout` consumer with its OWN `useState`; Provider wraps a single `<div>` child to respect the single-child-Provider limit) + `routes/context.tsx`. ### Slice 3 — dynamic `generateMetadata()` — DONE 2026-06-27 (⚠ UNCOMMITTED) — HEAD API COMPLETE
Per-request `generateMetadata(props)` (sync + async) runs under the warmed QuickJS pool; its result populates the served `<head>`. Mirrors the RSC/error-loading-boundary "boot-plan + per-request pool render + serve injection" pattern.
- **Engine** (`src/runtime/quickjs_engine.rs`): `__ALBEDO_EVAL_METADATA(entry, propsJson)` script (looks up `__ALBEDO_MODULES[entry].generateMetadata`, calls it, awaits if thenable, returns `{ok, value:<object|null>}` — value is DATA not HTML) + `QuickJsEngine::eval_route_metadata(entry, props_json) -> Option<Value>` (inherent pub method; `with_engine` hands a concrete `&mut QuickJsEngine`, so no `RuntimeEngine` trait change). - **Build** (`src/manifest/builder.rs`): `detect_dynamic_metadata(root_component)` = `module.functions.contains_key("generateMetadata")` → `Some(component.name)`. - **Shared lowering** (`src/manifest/metadata.rs`, all `pub`): `render_head_metadata(route, &RouteMetadata)` (the title+desc+meta block, with `ALBEDO {route}` fallback — used by BOTH build_shell static path and serve dynamic path, so byte-identical) + `lower_metadata_object(&Value)` (generateMetadata JSON → RouteMetadata, wraps the private `metadata_from_json`) + `const DYNAMIC_HEAD_MARKER`.
