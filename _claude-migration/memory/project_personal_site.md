**⚠ Verification gotcha (important for next session):** driving the scroll in the preview via `window.lenis.scrollTo(x,{immediate:true})` does NOT emit Lenis's `scroll` event, so `scrollState.film` stays stale and nothing animates — looks broken but isn't. Use **real `wheel` events** (`window.dispatchEvent(new WheelEvent('wheel',{deltaY,bubbles:true}))`) to actually advance the film. Also: long `setInterval` wheel-loops inside a single `preview_eval` tend to **time out the harness (30s)** and can leave a runaway timer that pegs the renderer — if the preview goes unresponsive, `preview_stop`+`preview_start`+re-navigate. Prefer a short **synchronous** burst of a few wheel events that returns immediately, then a separate `preview_screenshot`.
